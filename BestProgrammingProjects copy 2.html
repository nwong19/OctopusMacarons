INFORM




This is my Inform Project called 'Lost in the Forest.' Inform is a coding language that revolves around text-based story-telling. This project is similar to a virtual game. The object is to arrive home and cure your sick mother. This was my first 'big' project and is still one of my favorite languages. This is an excerpt from the project that explains the story line and what the goal is. Although there is one 'main' goal, there are many obstacles that have to be solved before the main goal can be reached. This leads to trouble spots and loopholes that I had to fix in my code...




One of the components of my code was including a 'secret' room. This was difficult because this 'room' was actually a portal that led the player back home to their mother. However, if they had just choosen any direction to go and it happened to be this direction, they could finish the game without going through the nessecary steps. This also got rid of the 'challenge' element of the project. To solve this problem, I had to make an 'if' statement that said if the player was not carrying certain items, they could not go that direction.

Another bug that I had experienced was the stone that needed to be pushed to reach the key. I also had to learn how to make the stone an enterable supporter so the player could step on it.

SCRATCH




Scratch was a language where our class was first introduced to blocks. The project above is my final Project which is a Paleolithic Quiz. We had to relate this project to one of the topics we were learning in class so I tied mine with Social Science, which was the Paleolithic and Neolithic Age. I had a lot of fun with this project because after I got through the coding all I had to do was edit the sprites. I also added music, and bells that signified the music being on or off. There were some loopholes that I had to patch up, but I learned a lot from this project such as timing, broadcasts, and how to make comments on blocks. We also used blocks for an activity for Hour of Code, so this really helped me understand how each block works together to make a product.




One thing that was harder to fit together and make come out right were the monkeys. When you first open the quiz, there are two monkey that walk back and forth, never touching each other or the wall. First I had to see how many steps they had to take before they touched the wall, and then I had to coordinate them so they were facing eachother when they came towards the middle, and opposite direction when they were going towards the wall. I also had to make sure that they would restart every time the person went back to the home screen and that they were coordinated.

Another thing that I learned about was broadcast. This feature helps make other sprites seemingly 'disappear' or 'appear' when a certain broadcast is called out. Broadcast was also a very helpful feature because it allowed my questions to show up and hide when I wanted them to.

MEGGY JR




The Meggy Jr is actually just the machine that you see in the picture above. However, the code that I completed was actually on an app that I downloaded called Arduino. Connecting the Meggy Jr and Arduino was a long and grueling journey (just kidding), but it was worth it. I really liked the Meggy because it's like an actual game controller except for the fact that the game is actually on the controller as well. The project that I did is called "Final Meggy Project," which is a pretty explanatory title. This was a game that was difficult. A bug was that the player would move rather slowly, but I learned how to make it steady so that the player must pace themselves enough that they can reach the other side of the screen. One thing that I thought was very peculiar was that because I made the game, it was pretty easy for me to accomplish the games that I made. TIPS


Sometimes coding can feel like this:




Especially when things get stressful! It's important to take remember to take breaks if you've been coding for long amounts of time. Coding can be tricky when small things such as a space or a ; are out of place. As you work with coding more and more, it'll be easier to spot these ridiculous mistakes and improve on them. Trust me, the moment when you find the mistake and its a small misplaced ; or a /, it's one that will be priceless.



A lot of times this will happen:




It's important to know what your code is doing! Sometimes it can be hard to remember certain functions or understand why something is happening when you know it shouldn't be because you didn't code it! Make sure that you literally type out the code and not copy/paste because you can learn what things do what by typing it out. It's also very important to add comments to your code! Almost every single coding website will have a comment feature for their code so use it! This will help a lot if you're confused by your code.



And this...




Take your time, because when things are rushed, your computer will explode. Just kidding. But, when things are rushed, that's when mistakes happen.



There's a whole coding community!




There are a bunch of people who will help you with your code as well as many reliable sources on the internet. If you're hesitant about coding, you should start! It's an awesome experience that you can learn a lot from.
